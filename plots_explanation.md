# Объяснение графика `elbow_plot.png`

Файл `elbow_plot.png` генерируется автоматически при запуске `train.py` (функция `save_elbow_plot` в модуле `train.py`). График используется для обоснования выбора оптимального числа кластеров `K` в задаче кластеризации квартир (Задание 4).

---

## Что изображено

На графике две кривые на одной оси X (число кластеров `K` от 1 до 10):

### 1. Inertia (синяя линия, левая ось Y)

**Inertia** (она же WCSS — Within-Cluster Sum of Squares) — сумма квадратов расстояний от каждой точки до центроида её кластера:

```
Inertia = Σ_i || x_i − μ_{c(i)} ||²
```

Чем больше кластеров — тем меньше inertia (в пределе при `K = N` каждая точка становится своим кластером и inertia = 0). Поэтому смотреть на абсолютное значение бессмысленно — важен **темп убывания**.

### 2. Silhouette score (зелёная линия, правая ось Y)

**Silhouette score** — независимая метрика качества кластеризации, измеряющая, насколько объекты внутри кластера ближе друг к другу, чем к объектам соседних кластеров. Значение в диапазоне `[−1, 1]`:

- ближе к **+1** — кластеры компактные и хорошо разделены;
- около **0** — границы кластеров размыты;
- меньше 0 — объекты ошибочно отнесены к чужим кластерам.

Считается только для `K ≥ 2` (для одного кластера понятие «соседних кластеров» не определено), поэтому зелёная линия начинается с `K = 2`.

### 3. Красная вертикальная пунктирная линия

Отмечает значение `K`, выбранное автоматически как точка локтя (см. ниже).

---

## Метод локтя (elbow method)

Идея: при оптимальном `K` рост числа кластеров начинает давать всё меньше выигрыша по inertia. На графике это выглядит как «излом» — после которого кривая становится почти горизонтальной.

### Как ищется точка локтя в коде

Реализация в `train.py` — функция `find_elbow`:

1. Проводим воображаемую прямую линию от первой точки `(K=1, inertia[0])` до последней `(K=10, inertia[9])`.
2. Для каждой точки кривой считаем перпендикулярное расстояние от точки до этой прямой.
3. `K` с **максимальным расстоянием** = точка локтя.

Это классический геометрический способ найти точку наибольшего «провисания» кривой относительно прямой. Альтернативные методы — KneeLocator из `kneed`, второй дискретный дифференциал, gap statistic — но для данной задачи геометрический подход стабилен и не требует внешних зависимостей.

---

## Как читать график

1. **Начало кривой inertia крутое** — добавление каждого следующего кластера сильно уменьшает ошибку.
2. **В точке локтя угол меняется** — кривая выполаживается.
3. **После локтя добавление кластеров даёт малый прирост качества**, но усложняет интерпретацию модели.

Дополнительная проверка через silhouette:
- Если silhouette максимален в той же области, что и точка локтя — выбор `K` подтверждён.
- Если silhouette показывает другой пик — стоит присмотреться (возможны несколько разумных значений `K`).

## Почему именно эти признаки для кластеризации

В `train_clustering` для k-means используются:

| Признак         | Зачем                                                            |
|-----------------|------------------------------------------------------------------|
| `price_per_m2`  | Главный индикатор класса жилья — основа разделения на категории  |
| `area`          | Большая площадь часто коррелирует с премиум-сегментом            |
| `dist_to_center`| Близость к центру — фактор статусности                            |
| `district` (OHE) | Район — сильный категориальный сигнал                           |

Все признаки масштабируются через `StandardScaler` перед подачей в k-means: алгоритм основан на евклидовом расстоянии и без масштабирования крупные численные признаки (например, площадь) подавили бы вклад мелких (расстояние).

---

## Маппинг кластеров на классы

После обучения финального k-means с `K = K_optimal`:

1. Считается **средняя `price_per_m2` внутри каждого кластера**.
2. Кластеры **сортируются от дешёвого к дорогому**.
3. Первому присваивается имя `эконом`, далее по порядку: `комфорт`, `бизнес`, `премиум`.

**Если** оптимальное `K` **не равно 4** — используются обобщённые имена `класс 1`, `класс 2`, ..., `класс N` (отсортированные по той же логике). Маппинг сохраняется в `metrics.json` в поле `clustering.cluster_to_class`.

---

## Связь с теорией

- График визуализирует trade-off между **сложностью модели** (число кластеров) и **качеством разбиения** — аналогично trade-off bias/variance в обучении с учителем.
- Большое `K` = низкая bias, но высокая variance (модель «подгоняется» под отдельные точки).
- Малое `K` = высокая bias (объединяются разные сегменты в один кластер), но низкая variance.
- Локоть — компромиссная точка между этими крайностями.
