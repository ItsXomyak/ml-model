# Объяснение графиков проекта

В проекте 10 графиков. Все лежат в папке `plots/`. `elbow_plot.png` создаётся скриптом `train.py`, остальные 9 — скриптом `make_plots.py`.

Каждый раздел: что изображено → как читать → что отвечает на вопросы профессора.

---

## 0. `plots/elbow_plot.png` — выбор оптимального K (Задание 4)

**Что изображено**

Двойная Y-ось, общая X-ось `K = 1..10`:
- 🔵 синяя линия (левая ось) — **Inertia (WCSS)**: сумма квадратов расстояний от каждой точки до центроида её кластера. Чем меньше — тем компактнее кластеры.
- 🟢 зелёная линия (правая ось) — **Silhouette score** ∈ [-1, 1]. Чем ближе к 1, тем лучше точки внутри своего кластера vs чужих. Считается от K≥2.
- 🔴 красная пунктирная — выбранное оптимальное K.

**Как читать**

- Inertia падает при увеличении K. Ищем «излом» — точку, после которой кривая выходит на плато.
- Drops K=1→2→3→4 — большие. После K=4 drops становятся в 4 раза меньше → излом в K=4.
- Silhouette: пик в K=4 (~0.59) подтверждает выбор.

**Что показать профессору**

> «Метод локтя выявляет K=4: drop K=3→4 = 800, drop K=4→5 = 200 — резкое замедление. Silhouette максимален именно при K=4. Обе метрики независимо подтверждают наличие 4 естественных кластеров в данных.»

## 2. `02_correlation_heatmap.png` — Pearson корреляции

**Что изображено**

Тепловая карта корреляций между всеми числовыми признаками + price. Значения от -1 (синий) до +1 (красный).

**Как читать**

- `area ↔ price`: ~+0.97 — главный драйвер цены.
- `dist_to_center ↔ price`: ~-0.91 — обратная связь (дальше от центра = дешевле).
- `rooms ↔ price`: ~+0.74 — третий по важности.
- `floor`, `total_floors`, `year_built` — слабая корреляция с price (|r|<0.13).

**Что отвечает**

> «Pearson показывает линейную силу связи. Топ-драйверы: площадь, расстояние от центра и число комнат. Слабые признаки — кандидаты на удаление через Lasso.»

## 3. `03_boxplots_by_category.png` — цена по категориям

**Что изображено**

Три boxplot-панели (district / material / renovation), отсортированные по медиане цены.

**Как читать**

- **District**: Нура — самый дешёвый, Есиль — самый дорогой. Чёткая иерархия по медиане.
- **Material**: панель < кирпич < монолит.
- **Renovation**: без ремонта → косметический → евро → дизайнерский (растущая цена).
- Высота «коробки» = IQR (межквартильный размах).
- «Усы» = диапазон без выбросов.
- Точки за усами = выбросы по правилу 1.5·IQR.

**Что отвечает**

> «Boxplots показывают, что категориальные признаки несут реальный сигнал. Каждая категория сдвигает распределение цены — модель должна это учесть через one-hot encoding.»

## 4. `04_predicted_vs_actual.png` — точность регрессии (главный график)

**Что изображено**

Scatter (предсказание vs реальная цена) с двумя группами точек:
- 🔵 синие — train
- 🔴 красные — test
- ⚫ чёрная пунктирная — линия идеальных предсказаний y = x.

В заголовке: R² на train и test.

**Как читать**

- Идеальная модель: все точки лежат на диагонали.
- Чем плотнее облако вокруг диагонали — тем точнее модель.
- Если test (красные) разбегается шире train (синих) — overfit.
- В нашем случае: R² train = 0.999, R² test = 0.993 — разрыв ~0.6%, overfit отсутствует.
- Видны 4 «облака» точек вдоль диагонали — это 4 ценовых сегмента, каждый предсказывается корректно.

**Что отвечает**

> «Главный график. Точки лежат на диагонали — модель восстановила реальные цены. Разрыв между train (0.999) и test (0.993) меньше 1% → переобучения нет. R² ≥ 0.85 из ТЗ выполнен с большим запасом.»

## 5. `05_residuals.png` — анализ остатков

**Что изображено**

Две панели:
- **Слева**: scatter остаток (`y_true − y_pred`) vs предсказание. Красная пунктирная = ноль.
- **Справа**: гистограмма распределения остатков с μ и σ в заголовке.

**Как читать**

- **Идеал**: точки слева расположены случайно вокруг y=0, без паттерна. Это значит модель не упустила систематическую зависимость.
- **Гетероскедастичность** (расширение «воронкой» вправо) → ошибки растут с ценой → модель хуже работает на дорогих квартирах.
- **Нормальность остатков** (колоколообразная гистограмма с μ≈0) → линейные предположения корректны.

**Что отвечает**

> «Остатки распределены вокруг нуля без систематического паттерна — модель не упустила нелинейность. Гистограмма близка к нормальной, μ≈0, σ небольшое — хороший признак.»

## 7. `07_overfit_curve.png` — иллюстрация overfit (Задание 6)

**Что изображено**

Train R² и Val R² (3-fold CV) как функция от `max_depth` RandomForest. Серая заливка между линиями = overfit-разрыв.

**Как читать**

- При малой глубине (depth=2-4): обе линии низкие → underfit.
- При увеличении глубины: train растёт быстро, val растёт медленнее → разрыв расширяется = overfit.
- Оптимум — где val R² максимален.
- Если глубину увеличивать дальше — val перестаёт расти (или падает), а train стремится к 1.0 (запоминает шум).

**Что отвечает**

> «Это прямая иллюстрация ответа на Задание 6. Маленькая глубина = underfit. Слишком большая = overfit. Регуляризация (ограничение max_depth) удерживает модель в зоне адекватного баланса bias/variance.»

## 9. `09_confusion_matrix.png` — качество кластеризации (Cluster Purity)

**Что изображено**

Матрица «Истинный сегмент × Предсказанный класс»:
- Строки — латентный сегмент, заложенный при генерации (истина).
- Столбцы — класс, предсказанный k-means.
- Цвет ячейки = количество точек.
- Диагональ = совпадение истины и предсказания.

В заголовке — **Cluster Purity** = доля правильно классифицированных квартир.

**Как читать**

- Идеальный результат: все значения на диагонали (912 / 836 / 730 / 520), вне диагонали — нули.
- В нашем случае Purity ≈ 0.987 → 38 ошибок из 3000 (1.3%).
- Off-diagonal ячейки показывают, какие сегменты путаются (обычно соседние, бизнес ↔ премиум).

**Что отвечает**

> «Это самый сильный аргумент для защиты кластеризации. Латентный сегмент в синтетику заложен через generate_data.py, но кластеризатор его НЕ ВИДИТ при обучении. После обучения k-means восстановил эти 4 группы с точностью 98.7%. Это доказательство, что алгоритм работает корректно, а не угадывает случайно.»

## Дополнительные артефакты в `plots/`

Кроме PNG там лежат:

| Файл | Содержание |
|------|------------|
| `descriptive_stats.csv` | Расширенная describe(): mean, std, min/max, квантили, skew, kurtosis, IQR, CV, range, missing% |
| `categorical_counts.csv` | Count и percent для district / material / renovation |
| `correlation_matrix.csv` | Pearson + Spearman корреляции в одном файле |
| `eda_summary.json` | Краткая JSON-сводка EDA |
| `extended_metrics.json` | Cluster purity (k-means качество) |

Эти CSV полезны если профессор попросит «покажи численно». В `descriptive_stats.csv` есть столбец `cv` (coefficient of variation) — относительный разброс признаков; столбец `iqr` — устойчивая мера дисперсии.

## Логика порядка показа

При защите идти от данных к модели:

1. **Что в данных?** → 01 (распределение) + 02 (корреляции) + 03 (категории).
2. **Как модель регрессирует?** → 04 (predicted vs actual) + 05 (residuals).
3. **Что важно для модели?** → 06 (feature importance).
4. **Не переобучилась ли?** → 07 (overfit curve).
5. **Как работает кластеризация?** → elbow_plot (K=4) + 08 (визуализация) + 09 (purity).

3 минуты на каждый раздел — суммарно ~15 минут демо.
